Telnetman Training エリート編

1. Work Flow の作成

はじめに

行程の長い作業を行うときはTelnetman Work Flow を使うと捗る。
ここでは使い方のみを解説するので、入門編から上級編までに作成したTelnetman データを各自適当に使って操作方法を覚える。
注意として、Work Flow の規模が大きくなるに従って開いた時の描画に時間がかかる。数分を要することもある。待っていれば開く。

1-1. Work Flow の新規作成

Work Flow のタイトルと編集パスワード、Work Flow の実行パスワードを定義して[作成]を押す。

編集パスワードを入力して[編集]ボタンを押す。

1-2. Start の定義

① Start をダブルクリック
② 「デフォルトのログイン情報」にTelnetman でダウンロードしたログイン情報のデータファイル「Telnetman2_loginInfo_****.json」をドラッグ&ドロップ。
③ 必要があれば「enable password」に特権モード移行パスワードを入力。

[更新]を押して保存。

1-3. work の作成

① [new work]をクリック
② 適宜ドラッグ&ドロップで位置や線の屈折、接続を調整
③ タイトルを付ける
④ 流れ図データファイルをドラッグ&ドロップ
   「Telnetman2_flowchart_****.json」

個別ログイン情報 : デフォルトのログイン情報を使わない場合は個別で設定する。
syslog設定、diff設定、任意ログ設定も必要であればドラッグ&ドロップ。

Telnetman のaction の機能でパラメーターシートを編集できるが、
この段階ではまだ元のパラメーターシートのファイルに書き込まれていない。
編集分を破棄するか、書き込んで次に渡すかどうか選択できる。

[更新]を押して設定保存。
このとき配置データも保存される。

1-4. Through

Through は流れ図実行を飛ばしたいとき、その飛ばし先を指定する。

1-5. case の作成

パラメーターシートの内容で場合分けできる。
書き方はTelnetman のaction - 分岐条件と同じ。
最後の分岐条件に「1」と書くとプログラムで言う「else」になる。

領域が狭いときは左上の[拡張]を押す。

1-6. terminal

Goal 以外で終了させたい所にはterminal を置く。

2. タスクの作成と実行

2-1. タスクの作成

対象Work Flow の右上のボタンを押してタスク一覧を表示する。
新規タスクのタイトルを入れて[作成]を押す。

2-2. タスクの実行

実行パスワードを入力して[実行]を押す。

2-3. Start

① Telnetman のログインIDとパスワードを入力
② Start をダブルクリック
③ パラメーターシートをドラッグ&ドロップ
   「Telnetman2_parameter_****.csv」

[start]を押す。

2-4. work の実行

① 緑色で塗りつぶされている所がパラメーターシートのある所。
   ダブルクリック
② 流れ図実行対象に✔
③ 対象機器のログインIDとパスワードを入力

[exec]を押すとTelnetman Work Flow が
Telnetman にアクセスして流れ図を実行する。

[through]を押すと✔ が入っていない機器のパラメーターシートが
Through の矢印の方向に移動する。
流れ図は実行されない。

実行後、
- 使用されたパラメーターシート
- OK ログ
- NG ログ
- Error ログ
- Diff ログ
- Optional ログ
をダウンロードできる。

既にパラメーターシートが存在する箱に新たにパラメーターシートを移動させるとどうなるか。
↓
パラメーターシートは上書きではなく合成される。

3. 手動OK NG

3-1. 空箱

編集画面

流れ図データを入れないwork つまり、空箱を設置する。

実行画面

手動でOK, NG を決められる。
目視で状況を判断したいとき、別のツールでオペレーションをしたいときなどに使う。
