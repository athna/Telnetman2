Telnetman Training 入門編

1. 基本操作

1-1. command 作成

タイトル : 何でも良い。
コマンド : show running-config
を新規作成。
これ以外はデフォルトのまま。

1-2. 流れ図作成

1-1 で作成したコマンドを流れ図の左上にドラッグ&ドロップ。

1-3. パラメーターシート

A2, A3, A4, A5, A6, … に対象機器のIP Address を記入。

1-4. ログイン情報

ユーザー名、パスワード、enable パスワードを記入。
それ以外は機器に合わせて適宜変更。
(cisco 機器ならデフォルトのままで良い)

1-5. 実行

実行ボタンを押す。
左のように結果が表示される。
クリックするとログが表示される。

2. Jumper

前置き

準備

コマンドをもう1つ作成する。何でも良い。
例) show version

2-1. Jumper の挿入

2つのコマンドの間にJumper を入れて実行してみる。
色々なJumper を入れてどう動くか確認してみる。

実験

Jumper を含めて色々な配置を試してみる。

3. OK, NG 分岐

前置き

例として、パラメーターシート記載のホスト名と対象機器のホスト名が
一致しているかどうか確認する流れ図を作成する。

準備

hostname を確認するcommand を作成する。

コマンド : show running-config | include hostname 

3-1. パラメーターシートで変数を定義

C1 : hostname
C2, C3, C4, C5, C6, … : 対象機器のホスト名
と記入して変数hostname とその値を定義する。
C列以降であればどの列でも良い。

3-2. action の作成

command で取得したhostname と
パラメーターシートのhostname を比較するaction を作成する。

タイトル : 何でも良い。
抽出パターン : hostname (.+)
分岐条件 : '{$1}' eq '{hostname}'
NGメッセージ : 何でも良い。

3-3. action を流れ図に配置

準備で作成したホスト名取得のcommand の下にホスト名確認のaction を配置する。
実行してOK 終了することを確認する。

実験

次に、左のようにパラメーターシートのhostname に誤った値を記入して実行し
NG 終了するか確認する。

4. 設定変更

前置き

現在のホスト名を確認(事前確認)
↓
ホスト名の変更
↓
新しいホスト名の確認(事後確認)

の流れ図を作成して実行してみる。

準備

パラメーターシートに変数new hostname を追加。
値は何でも良い。


分岐条件
'{$1}' eq '{hostname}'
↓
'{$1}' eq '{new hostname}'
に変更した事後確認用のaction をもう1つ作成する。
タイトルやNG メッセージも適当に変更する。


コピー機能を使うと楽である。

4-1. 設定変更command の作成

パラメーターシートのnew hostname の値で
ホスト名を設定するcommand を作成する。

系統 : conf t
conf t, end : する
コマンド : hostname {new hostname}

4-2. 設定変更の流れ図作成と実行

ホスト名取得
↓
現在のホスト名の確認
↓
ホスト名変更
↓
ホスト名取得
↓
新しいホスト名の確認

の順に並べる。
実行してみてホスト名が変わり、
OK 終了することを確認する。

実験1

続けてもう1度実行すると、ホスト名変更前にNG 終了することを確認する。

実験2

変数名のみを入れ替えてから、もう1度続けて実行するとホスト名が元に戻り
OK 終了することを確認する。
